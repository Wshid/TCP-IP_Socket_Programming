UDP : 신뢰성 측면에서 TCP보다 낮지만, 보다 빠른 전송이 가능함

UDP와 TCP의 큰 차이 : 흐름제어(Flow Control)
    UDP는 TCP보다 구현이 용이하며,
        데이터의 손실이 크게 발생하는 것이 아니기 때문에
            신뢰성 < 성능 일때, UDP가 TCP보다 좋음

UDP 역할
한 host로 패킷 도착 시,
    해당 패킷을 정확한 UDP 소켓(포트)에 전달

압축 파일 : TCP 사용
    구조상, 구성요소중 하나라도 손실시, 압축의 해제가 어려움

멀티미디어 데이터 : UDP 사용
    실시간 서비스를 지원
    한, 두개 손실되어도 크게 문제가 되지 않음


TCP가 UDP보다 느린 이유
    데이터 송수신 이전, 이후에 거치는 연결 / 해제 과정(handshaking)
    데이터 송수신 과정에서 거치는 신뢰성보장을 위한 흘
